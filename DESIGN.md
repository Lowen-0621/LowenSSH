# LowenSSH 设计系统

> AI SSH 智能运维 Agent 的界面设计源（source of truth）。桌面端与 CLI 共用这一套设计 token 与事件语义规则。

## 1. 设计 thesis（一句话）

深色运维驾驶舱：等宽字体打底，用终端蓝/琥珀/绿/红的状态语言，让「模型在思考 / 要跑命令 / 命令出结果 / 命令被拦截」四种状态一眼可辨。其中**安全拦截是全场视觉权重最重的事件**——这是 LowenSSH 的核心卖点（手写 agentic loop + 安全门禁），界面必须让「危险命令被拦住的那一下」砸下来。

记忆点（用户第一眼记住的东西）：**看得见 AI 在干什么，也看得见安全护栏起作用。**

## 2. 为什么深色优先

运维场景默认深色：① 终端/SSH 的天然语境，工程师肌肉记忆；② 长时间盯屏护眼；③ 让语义色（蓝/琥珀/绿/红）在深底上对比更强，状态识别更快。不做浅色主题（MVP 不需要，避免双套色板维护）。

## 3. 配色 token

```css
/* 底色层 */
--bg:        #0d1117;  /* 终端底 */
--surface:   #161b22;  /* 卡片 / 消息块 */
--surface-2: #1c2128;  /* 嵌套块 / 代码块 */
--border:    #30363d;  /* 分隔线 / 卡片边框 */

/* 文本层 */
--text:      #e6edf3;  /* 主文本 */
--muted:     #8b949e;  /* 次要文本 / 时间戳 / 提示符 */

/* 事件语义色（核心：6 类 SSE 事件各有其色） */
--model:     #58a6ff;  /* 蓝   token        —— 模型在说话 */
--tool:      #d29922;  /* 琥珀 tool_call    —— 模型要跑命令 */
--ok:        #3fb950;  /* 绿   tool_result  —— 命令成功执行 */
--danger:    #f85149;  /* 红   blocked      —— 命令被安全门禁拦截（最重） */
--error:     #db61a2;  /* 品红 error        —— 系统/连接错误 */
--ok-dim:    #238636;  /* 绿(暗) 成功状态的边框/底纹 */

/* 危险事件专用染色（blocked 卡片背景，红色低透明叠底） */
--danger-bg: rgba(248, 81, 73, 0.12);
--tool-bg:   rgba(210, 153, 34, 0.10);
```

色值取自 GitHub Dark 调色板：工程师熟悉、对比度经过验证、无需自己调可访问性。

## 4. 字体

```css
/* UI / 正文：系统字体，零加载成本 */
--font-ui: -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;

/* 命令 / 命令结果 / 终端内容：全部等宽 */
--font-mono: ui-monospace, "SF Mono", "Menlo", "Cascadia Code", "Consolas", monospace;
```

规则：**凡是「服务器上跑的东西」（命令、stdout、stderr、文件内容、日志）一律等宽**。模型的自然语言说明用 UI 字体。这条区分本身就在帮用户区分「人话」和「机器话」。

## 5. 间距与圆角

- 间距基准：8px 网格（4 / 8 / 12 / 16 / 24 / 32）。
- 圆角：图形版 `8px`（产品感）；终端版 `0`（锐角更像真终端）。
- 行高：正文 1.6，等宽内容 1.5。

## 6. 六类事件的可视化规则（最关键）

后端通过 SSE 推 6 类事件，前端必须让它们**互相区分、且 blocked 最突出**。

| 事件 | 语义 | 颜色 | 图形版渲染 | 终端版渲染 |
|------|------|------|-----------|-----------|
| `token` | 模型增量文本 | `--model` 蓝 | 左侧助手气泡，文字流式追加 | 直接追加到 log 流，蓝色文本 |
| `tool_call` | 模型要执行命令 | `--tool` 琥珀 | 琥珀标题卡片 + 等宽命令代码块 | `$ ` 提示符 + 琥珀命令行 |
| `tool_result` | 命令执行结果 | `--ok` 绿（executed=true） | 结果卡，绿色 ✓ 标头 + 等宽输出 | 缩进等宽输出块 |
| `blocked` | **命令被门禁拦截** | `--danger` 红 | **红色卡片 + 左红边框 4px + `--danger-bg` 染底 + ⛔ 图标 + 命令 + 拦截原因**，全场最重 | 整行 `--danger-bg` 红底：`[BLOCKED] <command> — <reason>` | 
| `done` | 任务最终结论 | `--text` + 绿点 | 收尾卡片，与普通消息区隔（顶部细分隔线 + 「任务完成」标记） | `── done ──` 分隔 + 结论文本 |
| `error` | 系统/连接错误 | `--error` 品红 | 品红条带卡片 | 整行品红 `[ERROR] <message>` |

**blocked 的设计硬要求**：它不能只是「换个颜色的普通卡片」。要有左边框、染底、图标三重强调，让用户扫一眼消息流就能发现「这里有个危险操作被挡了」。这是演示这个项目时最该被一眼看到的东西。

## 7. 两种界面形态对比

两者消费**同一个** SSE 端点（`POST /api/agent/stream`），共用同一套事件解析 composable（`useAgentStream`），只在渲染层不同。

| 维度 | 图形版（`/`） | 终端版（`/terminal`） |
|------|--------------|---------------------|
| 定位 | 「这是个产品」 | 「这是个工程师的工具」 |
| 布局 | 顶部连接表单卡 + 消息流（气泡 + 卡片混排） | 顶部仿 `ssh user@host` 输入行 + 单列 log 流 |
| 连接输入 | 表单字段（host / port / user / password / task） | 命令行式输入，回车提交 |
| 滚动 | 平滑滚动到底 | 跟随式 tail，新行自动贴底 |
| 圆角 | 8px | 0 |
| 适合 | 给非纯技术观众演示、截图 | 给工程师看、强调「真·终端」气质 |

顶部提供视图切换入口（图形 ⇄ 终端），同一会话能切换看两种呈现。

## 8. 反 AI-slop 约束（自我要求）

- 不用紫色渐变、不用居中堆叠、不用三列图标网格、不用装饰性色块。
- 不引入重型 UI 组件库——它们会盖掉这套精心设计的事件语义色，且对一个 6 类事件的对话界面是杀鸡用牛刀。手写组件，保持设计控制权。
- 密码字段做遮挡输入；明文密码只在内存中短暂存在，落盘一律 AES-GCM 加密，不留明文。

## 9. 安全相关的前端约束

- 表单里的 `password` 字段：`type="password"`、`autocomplete="off"`、不写入 localStorage/sessionStorage。
- 不在前端日志（console）打印密码与完整请求体。
- blocked 事件要把「拦截原因」完整展示——这正是安全门禁价值的体现，是功能不是噪音。
